# Итоговое задание по дисциплине «Тестирование прикладного ПО»

Репозиторий содержит полное решение:
- ручное тестирование (5 тест-кейсов);
- 2 баг-репорта по разным дефектам;
- Selenium-автотесты на каждый дефект;
- GitHub Actions CI, который запускает тесты и падает (красная сборка) из-за найденных дефектов.

## Структура

- `manual/test-cases.md` — ручные тест-кейсы.
- `bugs/BUG-001.md` — баг-репорт: ошибка граничного условия баланса.
- `bugs/BUG-002.md` — баг-репорт: проверка средств берётся не из выбранной валюты.
- `tests/selenium/test_transfer_defects.py` — автотесты на дефекты.
- `.github/workflows/selenium-ci.yml` — CI-пайплайн.

## Локальный запуск автотестов

1. Установить зависимости:
```bash
python3 -m pip install -r requirements.txt
```

2. Запустить приложение из `dist`:
```bash
python3 -m http.server 4173 --directory dist
```

3. В другом терминале запустить тесты:
```bash
BASE_URL=http://127.0.0.1:4173 pytest tests/selenium -q
```

Ожидаемый результат: тесты падают, так как проверяют корректное поведение, которое нарушено дефектами.

## CI

Workflow запускается на `push` и `pull_request`:
- поднимает статический сервер для `dist`;
- запускает Selenium-тесты;
- завершает job со статусом `failed` из-за непроходящих тестов.
